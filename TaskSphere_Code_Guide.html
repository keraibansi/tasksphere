Project Overview

TaskSphere is a full-featured SaaS-style task management web application built entirely with vanilla HTML, CSS, and JavaScript — no frameworks, no backend, no build tools required.

🎬

Animated Onboarding

3-slide intro with floating orbs, bouncing icons, and rocket animation. Shown once on first visit.

🔐

Multi-user Auth

Login & Register system with session management via LocalStorage. Includes social auth simulation.

📋

Full Kanban Board

4-column drag-and-drop board: To Do → In Progress → Review → Done (SortableJS).

👥

Team Collaboration

Invite members, assign tasks, view team activity feed and online/offline status indicators.

🤖

AI Task Suggestions

Type a project goal, get 4 smart task suggestions instantly. Click "Add" to put them on your board.

⚡

Real-time Sync Sim

Heartbeat interval simulates live collaboration: sync pulses, team notifications, status updates.

📊

Analytics Charts

Doughnut (status), Bar (categories), Line (7-day productivity) — all powered by Chart.js.

💳

Pricing Page

3-tier pricing (Free/Pro/Enterprise) with Monthly ↔ Annual billing toggle and live price updates.

✅ How to Open Simply double-click index.html — no server, no npm, no build step needed.

index.html — The Structure

The HTML file is divided into 6 major blocks, all inside <body>. They stack on top of each other — JavaScript decides which one is visible at any time.

Block 1 — Onboarding Overlay

<div id="onboarding-overlay">
  <div class="onboarding-wrapper">
    <div class="onboarding-slide active" data-slide="0"> ... </div>
    <div class="onboarding-slide"        data-slide="1"> ... </div>
    <div class="onboarding-slide"        data-slide="2"> ... </div>

The data-slide attribute tells JavaScript which slide is which. Only the slide with .active is visible. Each slide has its own CSS animation (orbs, feature boxes, or rocket).

Starts class="hidden" — revealed by JS after onboarding. Two panels share the same card; JavaScript toggles their .active class to switch between Login and Register.

Applied to the sidebar, modals, auth card, stat cards, and chart panels. The backdrop-filter: blur() is what creates the signature frosted-glass look.

Sidebar Collapse — CSS Sibling Trick

.sidebar { width: 260px; transition: width 0.3s; }

When JS adds .collapsed to the sidebar, CSS automatically shifts the main content to fill the extra space — no JS calculation needed.

script.js — The Logic

1. State — Data in Memory & LocalStorage

// Load from localStorage (persists across browser sessions)
let users     = JSON.parse(localStorage.getItem('ts_users'))   || defaultUsers();
let currentUser = JSON.parse(localStorage.getItem('ts_user'))  || null;
let tasks     = JSON.parse(localStorage.getItem('ts_tasks'))   || [];
let members   = JSON.parse(localStorage.getItem('ts_members')) || defaultMembers();

// Save back to localStorage after every change
function saveTasks() {
  localStorage.setItem('ts_tasks', JSON.stringify(tasks));
  renderAll();   // Rebuild all views
  updateCharts(); // Redraw charts
}
📖 How LocalStorage Works It only stores text strings. JSON.stringify() converts objects/arrays to text. JSON.parse() converts them back. Data survives browser close/reopen — zero backend needed.

2. Authentication System

// LOGIN — find user in stored array
document.getElementById('login-btn').addEventListener('click', () => {
  const email = document.getElementById('login-email').value;
  const pass  = document.getElementById('login-pass').value;
  const user  = users.find(u => u.email === email && u.password === pass);

  if (!user) { showToast('Invalid credentials', 'error'); return; }
  loginUser(user);       // Save session, show dashboard
});

// REGISTER — add new user to array
const newUser = { id: Date.now().toString(), name, email, password: pass, plan: 'Free' };
users.push(newUser);
localStorage.setItem('ts_users', JSON.stringify(users));
loginUser(newUser);

// LOGOUT — clear session
function logout() {
  currentUser = null;
  localStorage.removeItem('ts_user');
  // show auth screen again
}

3. Task CRUD — Create, Read, Update, Delete

// CREATE or UPDATE — one handler, identified by hidden task-id field
document.getElementById('task-form').addEventListener('submit', (e) => {
  const id = document.getElementById('task-id').value;  // empty = create

  const task = {
    id: id || Date.now().toString(),   // Unique ID from timestamp
    title:    document.getElementById('task-title').value,
    dueDate:  document.getElementById('task-date').value,
    priority: document.getElementById('task-priority').value,
    status:   document.getElementById('task-status').value,
    // ... more fields
    createdAt: new Date().toISOString()
  };

  if (id) {
    tasks = tasks.map(t => t.id === id ? task : t);  // UPDATE
  } else {
    tasks.push(task);                                  // CREATE
  }
  saveTasks();
});

// DELETE — remove from array by ID
function deleteTask(id) {
  tasks = tasks.filter(t => t.id !== id);
  saveTasks();
}

// TOGGLE COMPLETE
function toggleStatus(id) {
  const t = tasks.find(t => t.id === id);
  t.status = t.status === 'completed' ? 'pending' : 'completed';
  saveTasks();
}

4. Template Rendering — Objects → HTML

function createTaskItem(task) {
  const done = task.status === 'completed';

  return `
    <div class="task-item ${done ? 'completed' : ''}" data-id="${task.id}">
      <input type="checkbox" ${done ? 'checked' : ''}
             onchange="toggleStatus('${task.id}')">
      <h4 class="task-title">${sanitize(task.title)}</h4>
      <span class="task-tag tag-${task.priority}">${task.priority}</span>
      <button onclick="openEditModal('${task.id}')">Edit</button>
      <button onclick="deleteTask('${task.id}')">Delete</button>
    </div>`;
}

// Render all tasks
function renderTaskList() {
  const filtered = tasks.filter(/* apply filters */);
  document.getElementById('task-list-all').innerHTML =
    filtered.map(t => createTaskItem(t)).join('');
}
⚠️ Security: sanitize() All user text is run through sanitize() before injection into innerHTML. This escapes <, >, & to prevent XSS (Cross-Site Scripting) attacks.

5. SPA Router — Page Switching

document.querySelectorAll('.nav-item').forEach(item => {
  item.addEventListener('click', () => {
    // 1. Update active nav highlight
    document.querySelectorAll('.nav-item')
      .forEach(n => n.classList.remove('active'));
    item.classList.add('active');

    // 2. Show the correct view section
    const target = item.dataset.target;  // e.g. "kanban-view"
    document.querySelectorAll('.view-section').forEach(s => {
      s.classList.toggle('active', s.id === target);
      // toggle(name, force): adds if force=true, removes if false
    });
  });
});

No page reload, no URL change. Pure CSS class toggling. The CSS rule .view-section { display:none } and .view-section.active { display:block } do all the work.

function setTheme(theme) {
  document.documentElement.setAttribute('data-theme', theme);
  // document.documentElement is the <html> tag
  // Setting data-theme="dark" activates ALL [data-theme='dark'] rules
  localStorage.setItem('ts_theme', theme);
}

document.getElementById('theme-toggle').addEventListener('click', () => {

Feature 2 — Kanban Drag & Drop (SortableJS)

function initKanbanSortable() {
  ['pending', 'inprogress', 'review', 'completed'].forEach(status => {
    new Sortable(document.getElementById(`kanban-${status}`), {
      group: 'kanban',     // Cards can move between groups of this name
      animation: 180,      // ms for smooth drag animation
      ghostClass: 'sortable-ghost',  // CSS class for the placeholder

      onEnd(evt) {
        const taskId   = evt.item.getAttribute('data-id');
        const toStatus = evt.to.closest('.kanban-col').dataset.status;
        const task = tasks.find(t => t.id === taskId);
        task.status = toStatus;   // Update data
        saveTasks();               // Save + re-render
      }
    });
  });
}

Feature 5 — Analytics (Chart.js)

function updateCharts() {
  // Count tasks by status
  const done    = tasks.filter(t => t.status === 'completed').length;
  const pending = tasks.filter(t => t.status === 'pending').length;

  // Destroy old chart first (prevents "ghost chart" bug)
  if (statusChart) statusChart.destroy();

  statusChart = new Chart(ctx, {
    type: 'doughnut',
    data: {
      labels: ['Done', 'Pending', 'In Progress', 'Review'],
      datasets: [{ data: [done, pending, inProg, review],
        backgroundColor: ['#10b981','#f59e0b','#6366f1','#8b5cf6'] }]
    },
    options: { responsive: true, cutout: '65%' }
  });
}

Feature 6 — Calendar

function renderCalendar() {
  const firstDay    = new Date(year, month, 1).getDay();  // 0=Sun, 6=Sat
  const daysInMonth = new Date(year, month + 1, 0).getDate();  // e.g. 28

  // Empty padding cells before day 1
  for (let i = 0; i < firstDay; i++)
    html += '<div class="calendar-day empty"></div>';

  // One cell per day
  for (let d = 1; d <= daysInMonth; d++) {
    const dateStr  = `${year}-${month+1}-${d}`;
    const dayTasks = tasks.filter(t => t.dueDate === dateStr);

    // Coloured dots for each task's priority
    const dots = dayTasks.map(t =>
      `<div class="day-dot dot-${t.priority}"></div>`
    ).join('');

    html += `<div class="calendar-day" onclick="jumpToDate('${dateStr}')">
      ${d}
      <div class="day-indicators">${dots}</div>
    </div>`;
  }
}

Data Flow Diagram

Every user action follows this exact sequence:

User Action
→Click button, fill form, drag card
JS Event Handler
→addEventListener fires
Modify tasks[]
→push / map / filter on the in-memory array
localStorage.setItem()
→Persist to browser storage (survives refresh)
renderAll()
→Rebuild ALL views from the updated array
updateCharts()
→Destroy + recreate Chart.js charts
renderCalendar()
→Rebuild calendar with new task dots
showToast()
→Animated notification appears
💡 This is Unidirectional Data Flow Data lives in tasks[]. The UI is always built FROM the data (never the other way around). This is the exact same pattern used by React and Vue — you've implemented it in vanilla JS!
